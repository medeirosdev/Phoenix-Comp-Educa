App Bluetooth e Comandos BLE

Como o celular se comunica com a Bia, e todos os comandos que você precisa saber.

O App Phoenix

A Bia (PHX-1) é controlada via Bluetooth Low Energy (BLE) usando o app Phoenix disponível para Android. O app envia comandos de texto terminados em \r e recebe respostas do robô em tempo real.

O app possui as seguintes telas principais:

  • Conexão: scan de dispositivos BLE, conectar/desconectar
  • Dashboard: visão geral do estado, acesso rápido
  • PID: sliders para Kp, Kd, velocidade base
  • Calibração: sensores e giroscópio
  • Corrida: botões START e STOP
  • Linha Virtual: parâmetros do Pure Pursuit
  • Telemetria: gráficos em tempo real
  • Terminal: envio de comandos manuais

Protocolo de Comandos

Todos os comandos são strings de texto terminadas com \r. Exemplos: SST\r, PP,1.500\r.

Módulo 02

Modos de Operação e Firmware

Como o firmware da Bia está organizado, os modos de corrida e por que o giroscópio sempre precisa ser calibrado.

Arquitetura do Firmware

O firmware da Bia é escrito em C++ com framework Arduino rodando num ESP32-S3. Ele é organizado em módulos independentes, cada um responsável por uma parte do sistema:

  • bluetooth / user_interface — recebe comandos BLE, decodifica e chama os módulos corretos
  • line_sensors — lê os 9 sensores frontais, calcula posição e faz calibração
  • motores — controla PWM dos motores, aceleração progressiva
  • pid_controller — implementa o controlador PD com Kp, Kd
  • giroscopio_module — lê o IMU (MPU6050), integra heading
  • mapper_module — grava e armazena o mapa da pista
  • virtual_line — controlador Pure Pursuit
Separação de responsabilidades: cada módulo tem seu próprio .h e .cpp. Isso facilita desenvolvimento em equipe — uma pessoa pode trabalhar no PID enquanto outra trabalha no Pure Pursuit, sem conflitos.

Os Modos de Corrida

Line Follower (CLF)

O robô usa os sensores IR frontais para detectar onde está a linha em tempo real. O controlador PD calcula a correção e diferencia as velocidades dos motores a cada ~2ms. É o modo mais simples e robusto.

Virtual Line (CVL)

O robô segue um mapa pré-gravado da pista usando odometria (encoder + giroscópio). Em vez de reagir à linha física, ele se pergunta: "onde estou no mapa? para onde devo ir?" O algoritmo responsável é o Pure Pursuit. É mais complexo mas permite velocidades maiores pois "vê" a curva antes de chegar nela.

Line Chaser (CLC)

Modo de perseguição — o robô detecta e persegue outro robô à frente. Usado em modalidades de disputa.

Por que o Giroscópio Precisa Ser Calibrado Sempre?

O giroscópio (parte do IMU MPU6050) é um sensor inercial: ele mede a velocidade angular instantânea e integra o valor ao longo do tempo para estimar o ângulo (heading).

O problema: qualquer pequeno offset no valor "zero" do sensor se acumula ao longo do tempo (drift). Se ao ligar o robô o giroscópio não for zerado (GC), o heading ficará sempre errado — e toda a odometria do Virtual Line ficará incorreta desde o início.

Regra: sempre que ligar o robô para uma sessão de testes, execute GC com o robô completamente parado e nivelado. O firmware faz uma média de N amostras para calibrar o zero.

Aceleração Progressiva e ESC

Para evitar derrapagem na largada e estressar menos a mecânica, o firmware inclui um sistema de aceleração progressiva: ao receber SST, o robô aumenta gradualmente a velocidade até atingir a velocidade base configurada, em vez de largar na velocidade máxima instantaneamente.

O ESC (Electronic Speed Control) pode ser ativado para refinar ainda mais o controle de torque dos motores.

Módulo 03

Pure Pursuit e Linha Virtual

O algoritmo que permite ao robô seguir um mapa sem depender dos sensores de linha.

O que é Pure Pursuit?

Pure Pursuit é um algoritmo clássico de controle de caminho (path tracking) usado em robótica móvel e veículos autônomos. A ideia central é simples: defina um ponto alvo no caminho a uma distância fixa L_D à frente do robô, e calcule o ângulo necessário para alcançá-lo.

Como funciona na Bia

Etapa 1: Mapeamento

Antes de usar o Virtual Line, você precisa mapear a pista. O robô faz uma volta em modo Line Follower enquanto o módulo mapper_module grava um ponto a cada milímetro percorrido:

// Cada ponto no mapa tem:
struct TrackPoint {
    float x;            // posição X no plano (metros)
    float y;            // posição Y no plano (metros)
    float distance_m;   // distância acumulada desde o início
    float target_speed; // velocidade recomendada nesse ponto
};

Etapa 2: Odometria em Tempo Real

Durante a corrida com Virtual Line, o firmware estima continuamente a posição (X, Y) do robô usando:

  • Encoder: mede a distância percorrida
  • Giroscópio: mede o ângulo (heading) atual
// A cada ciclo (~2ms):
delta_dist = dist_atual - dist_anterior
robot_x += delta_dist * cos(heading_rad)
robot_y += delta_dist * sin(heading_rad)

Etapa 3: Busca do Ponto Alvo

O algoritmo varre o mapa a partir da posição atual e encontra o primeiro ponto a distância ≥ L_D do robô. Esse é o ponto alvo.

Etapa 4: Cálculo da Correção

// α = ângulo entre orientação do robô e direção ao ponto alvo
alpha = atan2(target_y - robot_y, target_x - robot_x) - heading

// omega = curvatura necessária (Pure Pursuit clássico)
omega = (2 * sin(alpha)) / L_D

// correção para os motores
correcao = ganho * omega

Parâmetros Chave

ParâmetroEfeito
VL (L_D)Look-ahead. Maior = trajetória mais suave mas reage mais tarde às curvas. Menor = reage mais rápido mas pode oscilar.
VG (ganho)Amplificação da correção. Muito alto = oscila. Muito baixo = não corrige suficiente.
VB (blend)Mistura Pure Pursuit com sensor de linha físico. 0.0 = puro PP, 1.0 = puro sensor. Útil para corrigir erros de odometria em retas.
VD (drift)Correção de drift: usa os sensores para "ancorar" a posição estimada à pista real.

Fluxo Completo — Virtual Line

1
GC → KOK — calibra giroscópio e sensores
2
CLF → COK → SST — mapeia a pista em modo Line Follower
3
SSP — para após completar a volta de mapeamento
4
Verifique o mapa via VC (deve mostrar N pontos razoável)
5
CVL → COK — muda para modo Virtual Line
6
Ajuste VL, VG, VB se necessário
7
SST — larga com Pure Pursuit ativo
Dica: comece com VD,0 (drift off) e VB,0.0 (puro PP) para estabilizar primeiro. Depois, reintroduza o blend e drift gradualmente para melhorar a performance.
